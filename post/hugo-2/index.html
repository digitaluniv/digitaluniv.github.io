ローカルで作成したWebサイトをGitHub Pagesに公開します。

GitHubアカウントの作成

https://github.com/

GitHubのアカウントを作成します。

アカウントの登録

ユーザ名、E-mail、パスワードを入力し、「Sign up for GitHub」をクリックするとプラン選択のページに移行します。

プランの選択

無料の「Free」プランを選択し、「Finish sign up」をクリックします。

無料のプランは公開されます。非公開にしたい場合は、月５～５０ドルのプランがありますので、そちらを選択してください。

メールアドレスの確認

新規登録が完了すると、「verify」のメールが送られてきます。

メール本文にあるURLをクリックして、メールを認証します。

Github Pagesへの公開

Github Pagesで静的Webページをインターネット上に公開します。

リポジトリの作成

GitHub Pagesで、ユーザのウェブページを公開するユーザサイトを作成します。

ページ右上の「+」アイコンをクリックし、「New repository」をクリックします。

「Repository name」に、「username.github.io」と入力し、「Create repository」をクリックします。usernameは、GitHubのユーザ名になります。

リーモートリポジトリの作成

ソース管理用のレポジトリをcloneします。

$ git clone https://github.com/username/username.github.io

cloneしたリポジトリに”HugoとGitHub PagesでWebページの公開（１）”で作成したソースコードを移します。

$ cp -p -f -R hugo_site/* username.github.io/

リモートリポジトリのpush

“HugoとGitHub PagesでWebページの公開（１）”で作成したpiblicディレクトリ配下のファイルをローカルのusername.github.ioへ移し、GitHubへpushするスクリプトをローカルWebサイトのディレクトリに作ります。

#!/bin/bash

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

# Build the project.
hugo

# Copy public
cp -p -f -R public/* ../username.github.io/

＃ Push repos.

cd ../username.github.io/

git add --all

msg="rebuilding site `date`"
if [ $# -eq 1 ]
then msg="$1"
fi

git commit -m "$msg"

git push origin master

シェルスクリプトファイルをprojectnameディレクトリにdeploy.shとして作成し、実行する権限を付与します。

chmod 755 deploy.sh

これを実行します。

$ ./deploy.sh
